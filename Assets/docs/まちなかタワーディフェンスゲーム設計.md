# ゲーム名
おのころ列島
厄災ウェーブ戦（タワーディフェンスゲーム）


## ゲームジャンル

まちづくりタワーディフェンス

## おおまかなゲームの流れ

プレイヤーはマップ空間内に防衛ユニットを配置する
配置フェイズが終わると攻撃ユニットによる襲撃が始まる
襲撃を耐えると報酬が得られる
得た報酬で防衛ユニットを増強する


## ゲームデザイン

Plateauの都市空間データを利用。
つぎつぎ起こる災害イベントに対して、地域、自助・共助・公助レベルをあげて対応していく。
公助（消防車など）が来るまで、共助（防火水槽、消防団など）で被害拡大を抑えつつ、自助（街頭設置消化器など）でいかに初動の被害を抑えられるか。
食料や貨幣にたいしてフローとストックのコントロールが重要。
災害で生産ユニットが破壊されたらフローが下がる。
生産ユニットをつくるにはストックが必要。
ステージが続くと災害イベント頻度と管理マップサイズが増えていく。


## タイムライン

明確な区切りはないが、ビルドタイムを終わらせて災害季節まで飛ばすことはできる。
平時（ビルドタイム）：プレイヤーが生産ユニットや防衛ユニットを配置する
インシデントタイム：ちょっとした襲撃などが発生する。平時の中で少しだけおきる襲撃。
災害の季節（ウェーブ）：本格襲撃が連続でくる。
報酬タイム：残った生産ユニットから報酬を得ることができストックすることができる。
マップの拡張：ゲーム初期は小さかった管理対象エリアを広くしていくことができる（広くなると管理が大変）


## インシデントイベント

冷夏：不作、森から動物達が攻めてくる
乾燥＆強風：火事が起きやすい
台風：-建物耐久、浸水
大地震：-建物耐久、火災大量発生
噴火：交通寸断、食料生産の中止
流行り病：病人が大量発生する
停電：産業活動の停止
戦争：複合災害
ゴジラ：カタストロフイベント
破局噴火：カタストロフイベント
